v3.1 → v3.2·展开态重设计

A
结构·一行 → 四区

v3.1 expanded 是·title 加粗 + body + detail + meta·四块都贴着上一块·没有视觉层级。v3.2 展开后结构变四区·(1) **头排**·title (15.5px 加粗) + 右上 turn 小 chip "T54·春"·下方 1px 金线·(2) **摘要**·40px chip-label "摘要" 在左 + 文字 12.5px·(3) **详情**·同结构·label color 用 type 色·(4) **底部**·meta 标签流 + 右下 "进入详情 ↗" type 色按钮。视觉层级清·像"卷宗摊开"。

B
ttype·9px 字头 → 24×24 chip (展开时)

折叠时 ttype 仍是 12px 单字 (议·军·势·etc.)。展开后 ttype 变 24×24 边框 chip·type 色边·14px 字·与 title 同行 align-start·像"卷宗封面的类章"。从"列表 prefix" 升级为"卷宗 anchor"。

C
stripe 加宽 2 → 4 px + type 色 halo

v3.1 stripe 不论折叠展开都 2px。v3.2·展开时 left stripe 加宽到 4px·且加 `-1px 0 6px var(--ttype-c)` 朱/钴/玉/etc. **halo**·让 stripe 像"印章浸朱漆"溢出一点·anchor 当前展开 item。

D
摘要/详情 chip-label·40×19 type 色边

v3.1 用 dotted divider 划摘要/详情。v3.2·两段各加 40×19 **chip-label** ("摘要" / "详情")·left 2px stripe = type 色·背景 dark·label 字 type 色·与展开整体 type-themed 一致。两段之间不再用 divider 划·靠 grid 排版自然分隔。

E
"进入详情 ↗" 按钮·type 色 chip

新增·展开底右 type 色 pill 按钮·hover 时 darker bg + 金光 + 字 brighter。点击后进游戏的全详情 panel (现 `openEventDetail(idx)`)。让 user 知道"展开是预览·真深读走详情"。

F
背景·type 色右下淡晕 + drop shadow 加重

v3.1 expanded bg 只一条 gradient wash。v3.2·加 type 色 ellipse 在右下 (`::after` opacity .12·非常淡)·和 stripe 一起形成左右对应的 type 染色·让人感觉 item 整个被 type 色"侵染"·而非只有左边一条 stripe。drop shadow 也加重·让展开 item 浮起·与折叠 item 形成 z 轴分层。

v2 → v3 改了什么

1
frame 进一步去·只剩 spine + 顶底 1px 金线

v2 还有 1px 金边轮廓 + 实底 .30 透明度 + backdrop blur。v3·**完全去 background base**·只 left 2px 朱漆 spine + 顶部 1px 金线 + 底部 1px 金线 (渐隐 gradient·像装裱条的边)·中间纯透·user 的御案桌面/地图直接透出来。框架不再"画出来"·只用三条线提示边界。

v2·spine + 金边 + 实底 .30 + blur(2) → v3·spine + 顶底 1px 金线·中间全透
2
chip SVG 全部砍掉

v2 每条 item 有 16×16 SVG chip (8 类 8 icon·虎符/官帽/邸报卷/etc.)。v3·**全部删**·用一个 9px 单字 ttype 替代 ("议/军/势/财/人/报/索/杂")·颜色仍按 8 type 区分但只染字色 (rgba .82-.86)·title 仍纯白金。SVG paradigm 在右侧栏 rail 已有 (那里是 entry·这里是 list)·没必要双份。

v2·16×16 8 SVG chip + 半透 stroke → v3·9px 单字 ttype·colored only
3
按回合分组·sticky turn header

v2 每条带 inline "T54·春"·重复 14 次。v3·把 turn 提到 group head·**所有同回合 item 共享一个 sticky header** "T 54·春" + 渐隐金线·往下滚滑 header 钉在顶。**符合古代邸抄真实形态** (邸抄每天一日·日下列事)·user 也能"按日扫读"。

v2·每条 inline "T54·春" 重复显示 → v3·sticky group header·一回合一段
4
head 加 filter + density toggle

v2 head 只 "邸报 三回合 14 ▾"。v3 head 升级·left "邸报 14条"·right **filter row** (全 / 朝 / 军 / 报 / ⋯) + **density toggle** (紧/宽)。filter chip 点了才高亮 + type 色边·"⋯" 折叠剩余 5 类。density·紧 = 22px·宽 = 28px·user 切换。

v2·单一 scope dropdown → v3·filter (8 类) + 紧/宽 切换
5
阅读态·已阅 item 淡化

新增·`item.is-read` class·title color 从 .92 降到 .46·ttype opacity .56·像古书已读条目自然褪色。**让 user 一眼区分"近 1-2 回合新事"vs"3 回合外旧档"**。class 自动加·当 item 离开当前 turn 即标记 read·实际 game 内 turn-based 推进即可。

v2·所有 item 视觉一致 → v3·is-read 淡化 .92 → .46
6
stripe 2px → 0·全靠字色染 type

v2 每条 item left 2px 朱/钴/玉/紫/etc. stripe (8 type 8 色)。v3·**stripe 删**·type 全靠 ttype 字色染 (9px"军"字本身染钴蓝色·"议"字染朱·"报"字染金·etc.)。少了一条竖线·更轻。盲扫·user 看色的字而非左竖线。

v2·left 2px stripe 8 色 + chip → v3·0 stripe·只 ttype 字本身染色
7
state·hot/new/alert 仅 6px 圆点 + title 微染

v2 已经把 marker 缩到 6px 圆点。v3 保留·但·**默认态完全 transparent (opacity 0)·只 hot/new/alert 才显**。v2 默认就显示一个灰点·v3 不显。圆点位置移到 right 14px col 居中·而非靠边。alert 仍 1.4s 脉冲·hot 静态·new 静态但 item 顶部 1px 闪光线 2.4s。

v2·6px 圆点·默认灰 → v3·6px 圆点·默认 hidden·只 state 显
8
list 内 padding / gap 全部砍

v2 list 还有 6px padding + 2px gap + dotted divider。v3 list padding 只 4px·item gap 0 (直接挨着)·**取消 dotted divider** (turn header 的渐隐金线已经够分隔)·item 之间靠 hover 才显边界。看起来真的像 "一栏文字"·而非"一堆 row card"。

v2·gap 2 + dotted divider → v3·gap 0·靠 turn header 分隔

v3 仍可调的方向

A·更极端·零 ttype·只标题

  • type 只靠"标题前缀字"区分 (如 title = "[军] 宣府总兵...")
  • 极简·完全文字流·无任何装饰元素
  • 缺点·type 识别度下降·需 user 自己读标题
B·sticky header 加日期
  • turn header 不仅 "T54·春"·加干支 "甲申年三月廿八"
  • 更"真邸抄"feel
  • + 加每个 turn 总条数 (T54·春·5条)
  • 缺点·header 占地略增·22 → 28px
C·item-level vrl turn 替代 sticky header
  • 放弃 sticky group header·改回 inline "T54·春" 在每条
  • 但 turn 用 vrl 朱印小印 (5×14 竖排) 替代横排
  • 视觉更"印章 + 朱抄"
  • 缺点·失去 sticky 滚动便利
