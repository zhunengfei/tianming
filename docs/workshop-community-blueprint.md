> 版本：2026-06-04 ｜ 状态：**preview 全量完成，未 port runtime** ｜ 用途：作为接入 0.5.0 服务器、逐步落地的纲领。

## 〇、一句话定位

天命的创意工坊**不是"又一个 Steam 工坊"**，而是一座**用 AI 演绎、由众人协作书写的活的历史**。
它的两条护城河（别家抄不动，因为前提是"一个有编辑器和 AI 演绎脑的中国史游戏"）：

1. **协作写史**：世界线树（分支宇宙）+ 史册接龙（玩 = 写史）。
2. **AI 原生工坊**：AI 自动试玩体检、装前试聊角色、AI 共创起草。

其余（社交、运营、治理、平台化）是把生态养活的常规引擎，价值在"把这两条做深"。

## 四、落地路线（建议分期）

> 原则：**先落能闭环、服务器现成的最小集，让玩家真用上**，再按护城河优先级逐步深做。别一次性全 port。

- **P0 · MVP 闭环（服务器基本现成）**
  port 内容商城的 **发现 / 浏览 / 详情 / 账号 / 评分 / 安装** 接 0.5.0（**剧本-only**）+ 新增 **评论**（一个小接口）。这是**可直接 ship** 的真闭环。客户端落点 `web/tm-content-manager.js`（保留已做的无障碍硬化），逐屏 playwright 验。
- **P1 · 多类型资源包**：立绘 / 音乐包 — 需二进制存储 + 配额 + 按类型 schema。**上线前先想清版权与存储成本。**
- **P2 · 社交层**：好友 / 私信 / 通知 / 盖楼 — 需新表 + presence infra。
- **P3 · 协作写史（护城河）**：Fork 血缘 / 世界线树 / 史册接龙 — 需 lineage + 史册上传 + 游戏内接龙转换。
- **P4 · AI 原生 + 治理 + 运营**：AI 自动试玩体检 · 信誉分 + 社区共治 · 运营看板 · 风控。
- **长期**：见第六节。

## 五、风险与成本（上线前必须想清）

1. **版权（玩家传立绘 / 音乐）**：最大雷区。需自动查重 + 上传门槛（仅认证作者）+ AI 生成标注 + 侵权下架闭环。
2. **存储 / 带宽**：大文件（立绘 / 音乐）在 owner **自托管**下成本陡增 → 配额 + CDN + 弱网降级 + 离线合集打包（中国网络现实）。
3. **审核不可扩展**：owner 人工把关在量大后必崩 → **信誉分免初审 + AI 预筛兜底 + 社区共治分级**是生死线。
4. **实时社交 infra**：好友 / 在线状态 / 私信比现 HTTP 服务器重，需轮询或 WS。
5. **并发写**：多桌面端 + 多 Claude 实例并行改文件曾自相矛盾 → 单写口 / 每次现读 / 版本号。

## 六、长期规划（深挖方向，记账备查）

> 这些是"再往后"的护城河深做与生态外延，非近期落地，但定方向时不该忘。

### 🏯 协作写史（已起头，可继续深做）
- **世界线树**全社区版（不止单剧本，整个朝代 / 跨朝代的分支宇宙）。
- **共笔长卷**：多作者协作写跨代 campaign。
- **史册接龙**做成正式玩法循环（游戏内"以史册为开局"）。

### 🤖 AI 原生工坊
- **AI 自动试玩 + 平衡体检**：每个投稿 AI 先跑 N 局→平衡 / 胜率 / 可玩性报告→再进人工审核（**同时解决质量与审核扩展**）。
- **装前试聊剧本里的 AI 角色**（极强转化钩子）。
- **AI 写无字封面提示词**→生成氛围底图，玩家审，汉字仍 DOM 叠（升级剪纸占位）。
- AI 标真实难度。

### 📜 历史即灵魂（点睛 + 合规护身）
- 信史 / 演义 / 架空三色标注 + 史料出处 + 考据分徽章。
- 历史词条百科（跨剧本 mini-wiki）。
- 史学背景认证（在创作者认证之上加一档）。
- 课堂 / 教育模式（远期垂直）。

### ⚙️ 生态引擎
- 冷启动流量池（对抗马太效应，扶持新作者）· 赛季 / 季主题 · 留存任务 · 创作者激励经济（非货币，配全平价）。

### 🛡️ 去中心化治理
- 信誉分驱动权限 · 社区共治分级初审（owner 终审）· 原创溯源（血缘 + 查重）· AI 生成内容标注。

### 🌐 平台化外延
- 离线合集打包 / CDN 镜像（中国网络）· 工坊开放 API / SDK · 跨设备云存档续玩 · 自托管镜像分发。
