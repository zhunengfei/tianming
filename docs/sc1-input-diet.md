# 第一刀施工依据 · sc1 输入瘦身（tp1 116K → 目标 ~75K）

> **日期**：2026-05-29
> **来源**：演绎脑实跑天启 T1（见 ai-relay-run-report-T1.md）+ 逐段尽职调查（读真实构建代码）。
> **定位**：第一刀（缩 sc1 输入体量）的可动手施工依据。每项都追到了构建代码、确认裁后 sc1 不缺推演依据。
> **数据基准**：天启 T1 的 sc1(#020) prompt = system 47K + user(tp1) **116676 字**。

---

## 0. 为什么是第一刀（vs 第二刀缓存）

- tp1(user) 116K 是成本与弱模型崩溃的**真正大头**，且**每回合变、无法缓存**。
- 第二刀（缓存）只能省 system 稳定段，且**依赖后端**（第三方中转多不支持）。
- **缩 tp1 直接省 input token、不依赖后端** → 治本，优先于第二刀的拆段/跨回合。

## 1. tp1 116K 解剖

| 区间 | 段数 | 合计 | 占比 |
|---|---|---|---|
| >2000 字（巨块） | 8 | 39.8K | 34% |
| 800-2000 | 21 | 26.2K | 22% |

| 100-300（碎块） | 130 | 22.3K | 19% |

**碎块(130×100-300字)无重复标题模式 = 异构规则/背景碎片**（很多是"【史料记载】出处 + NPC 背景"），逐个审计成本高、删错风险高，**第一刀不动碎块，主攻 8 个巨块**。

8 个巨块：军事力量 8.7K、"死亡名单"(实为输出 schema)12.5K、问对记录 5.2K、生命周期诏令 3.7K、史料记载 2.6K、势力间关系 2.4K、行政区划 2.4K、第5层记忆修正 2.3K。

## 2. 三类浪费

1. **重复喂**：问对对话在 sc1 prompt 注入**两次**（prompt.js:657 全文 + prompt.js:698 `_injectRecentDialogues` XML 汇总），且 sc1q 已结构化提取其承诺/决议/语气。
2. **静态数据每回合重塞**：史料记载/关键人物背景全是后金汉臣、蒙古/朝鲜边缘人物的《清史稿》出处 + 生平——整局不变、且本回合主推演用不上，却每回合塞进 tp1。
3. **粒度过细 + 专项已兜底**：军事(兵种/装备/年饷明细)、势力关系(全部两两关系)，sc18/sc16 各有独立完整数据源，sc1 主推演不需要这个粒度。

## 3. 尽职调查：逐项确认"裁后不缺依据"

### ① 问对全文（prompt.js:657-665）—— 删全文 + 补语气，依据不缺反去重
- **构建代码**：`tp += '召见X：玩家说"playerSaid"→NPC答"npcSaid"'`，直接塞 NPC 完整发言（5.2K 来源）。
- **双重注入**：657 全文 + 698 XML 汇总，同批对话喂两遍。
- **sc1q 已覆盖**：sc1q 输出 `dialogue_commitments`（承诺）+ `collective_resolutions`（决议）已注入 sc1 的【sc1q 推演输出】段。
- **语气方案（关键）**：sc1q **还提取了 `npc_dialogue_intent[{npc,mood,subtext,next_likely_move}]`（语气层），但当前没注入 sc1**（仅被 sc15n 消费）。→ **在 ai.js 的 sc1q 注入段补注入 npc_dialogue_intent（每 NPC 一行紧凑格式）**，则删全文后语气经 sc1q 无损保留。
- **裁后 sc1 仍有**：承诺+决议(已注入) + 语气(补注入) + 698 XML 汇总。**四来源裁最冗余的一个，依据不缺。**
- **落点**：删 prompt.js:657-665；补 ai.js sc1q 注入段（加 npc_dialogue_intent 紧凑行）。
- **收益**：5.2K → 几百字。**风险最低 + 自带去重红利。建议第一个做。**

### ② 军事明细（tm-ai-planning.js:810 blockF「军事力量——完整数据」）—— 砍配比保粗粒度
- sc1 军事相关输出（resource_changes / 武将 npc_actions）只需"军/兵力/士气/忠诚/统帅/驻地"粗粒度。
- 兵种配比、装备数量明细 → 只 sc18 战斗解算需要，**sc18 有独立完整数据源**（实跑 #030 自带兵种/装备/年饷）。
- **保留**：士气、忠诚（向背，有政治含义）、**欠饷**（关系"边卒哗变→兵变"政治推演，非纯军事细节）。
- **裁**：兵种配比、装备数量明细。
- **落点**：tm-ai-planning.js:810 blockF。**收益 8.7K → ~2K。低风险（sc18 兜底，sc1 大方向依据保留）。**

### ③ 静态人物背景（史料记载段，整局不变）—— 外移 sysP，按后端分策略
- 挪 sysP → 全管道可见（不会读不到，可见性反而扩大）。
- **缓存两约束**：(a) sysP 稳定段须字节稳定 → **不能在 sysP 内做每回合变的相关性过滤**；(b) 依赖后端支持缓存。
- **分策略**：支持缓存的后端(openai/anthropic 原生)→全量静态背景进 sysP 稳定段（缓存消化体量）；不支持的第三方中转→仍在 tp1 按相关性裁（接受不可缓存）。
- **与第二刀交汇**：静态背景从 tp1(全价/每回合) 移到 sysP 稳定段(可缓存) = 第一刀缩 user + 第二刀喂缓存，一举两得。

### ④ 势力关系（2.4K）、输出 schema（12.5K）—— 次轮
- 势力关系：只留与玩家相关 + 高烈度，sc16 兜底。
- 输出 schema：按 modelTier 真精简（char_updates 不列 12 个 delta），升级 ai.js:2942 的"提示少填"为"真给精简 schema 文本"。
