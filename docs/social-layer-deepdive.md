## 〇、总纲：五个方向不是五个补丁，是一套循环

社交软件的内核不是货架，是 **「内容流 × 关系 × 身份」的闭环**。天命独有的内容源是 **AI 演绎出的、独一无二的史册**——别去复刻探店种草，去把「玩家的崇祯三年」做成能刷、能聊、能接着写的内容。

```
            ┌─────────────────────────────────────────────┐
            │              个人资料库 · 史官档案            │  ← 脊梁（E + 你加点）
            │   称号/社区品级 · 列传(作品) · 战绩 · 收藏阁  │     一切活动在此沉淀为身份
            └───────▲───────────────▲──────────────▲───────┘
                    │               │              │
        声望沉淀     │      作品沉淀  │     战绩沉淀  │
                    │               │              │
   ┌────────────────┴───┐  ┌────────┴───────┐  ┌───┴──────────────┐
   │  A · 史馆动态流     │  │ B · 创作者生态 │  │ C · 接龙/世界线  │
   │  （内容层·地基）    │  │ （留住作者）   │  │ （护城河·独门）  │
   │  邸报体动态 + feed  │  │ 关注/三连/合集 │  │ 残局接演/同台竞史 │
   └─────────▲──────────┘  └───────▲────────┘  └────────▲─────────┘
             │                     │                    │
             └─────────────────────┴────────────────────┘
                                   │
                        ┌──────────┴──────────┐
                        │  D · 同好圈子（关系）│  把弱关系(下载)升成强关系(归属)
                        └─────────────────────┘
```

- **A 喂内容**（让玩法变成可刷的流）；**C 造独门内容**（fork 的活历史）；**B+E 造身份**（作者主页 + 档案 + 称号）；**D 造关系**（圈子归属）。
- **个人资料库是脊梁**：A/B/C/D 产生的每一次行为（发布、获赞、接龙、被收藏、升品）都回流到档案，沉淀成「你是谁」。没有这根脊梁，社交是一盘散沙。

### 与现状的接续（避免重复造）
runtime 现已有：好友(双向)/私信/通知/世界线树(lineage)/史册接龙(chronicles)/社区推荐(endorse)/评论/评分/作者页。**本案新增的是**：动态流(A，现无 feed)、关注图谱(B，现只有双向好友)、合集/约稿(B)、残局接演+同台竞史+共编(C 深化)、圈子(D)、**个人资料库(脊梁，现只有薄薄的「我-主页」)**。

## 一、信息架构（IA）：导航怎么变

现状顶栏：`发现 / 浏览 / 排行 / 专题 / 创作 / 好友 / 我`（7 项，逛与我混在一起）。

提议分两簇 + 新增三个顶级面（动态/圈子/档案）：

```
逛 ▸  发现   动态●   排行   专题            我 ▸  创作   圈子   好友   我的档案
      └ 货架  └ 内容流(A新)              └ 发布  └D新   └有    └ 档案库(E扩,脊梁)
```

- **动态(A)**：登录后的默认落地页候选——一打开看到的是「关注的人在演什么」，而非货架。
- **圈子(D)**：同好聚合。
- **我的档案(E)**：从「我」升级成完整资料库。
- 详情/接龙(C) 仍挂在内容浮层；作者主页(B) 仍挂在作者上（复用档案模板的公开视角）。

> 决策点①：动态(A)是否取代「发现」成为默认落地页？（小红书/抖音是 feed 优先；Steam 是货架优先。建议**登录态 feed 优先、未登录货架优先**。）

## 二、个人资料库 · 史官档案（脊梁，E + 你加点）

### 定位
把分散在各处的身份（成就 / 作品 / 战绩 / 收藏 / 称号 / 声望 / 关系）收进**一座馆**。参考 **Steam 个人资料 + 成就展柜 + B站个人主页 + GitHub profile 贡献图**。私有视角=「我的档案」，公开视角=「作者主页」(B)，**同一套模板两种可见性**。

### 模块（御案米金长卷，呼应人物图志列传范式）
```
┌──────────────────────────────────────────────────────────────┐
│ [印] 同志        〔进士·认证作者〕  关注 12 · 粉丝 340         │ 名册头
│      入坊 2026-05 · 末次 6-05      [私信] [关注] [举报]        │
├──────────────────────────────────────────────────────────────┤
│ 声望盘  ▓▓▓▓▓▓▓░░  进士 → 翰林  (声望 4820/6000)              │ ← 社区品级进度
│         发布·获赞·被荐·接龙·被关注 加权                       │
├──────────────────────────────────────────────────────────────┤
│ 列传(作品 6) ── 天启七年 ·盛唐立绘 ·边关配乐 …  [看全部 ›]    │ ← 我的发布(已有种子)
├──────────────────────────────────────────────────────────────┤
│ 战绩 ── 玩过 14 局 · 最久存续 崇祯12年 · 偏好 明末/硬核        │ ← 来自史册/通关回流

├──────────────────────────────────────────────────────────────┤
│ 功业(成就墙) ── 入籍·已发布·千人传抄·众望所归 …               │ ← 已有 .badges-wall
├──────────────────────────────────────────────────────────────┤
│ 履历热力图 ── ▢▣▣▢▢▣▣▣▢ (近一年活跃)                         │ ← GitHub 式贡献图
├──────────────────────────────────────────────────────────────┤
│ 题跋(留言墙) ── “收权节奏拿捏得好” —史海钩沉                  │ ← 别人给我的评价
```

### 社区品级体系（你加点的核心 + E）
**关键架构判断**：社区身份 = **玩家账号**层面，**不能**和游戏内角色的功名(`virtueMerit` 0~15000 / 18 品)混账——那是角色属性。但**可借科举功名的美学**（科举词跨朝代通用，不犯「UI chrome 朝代中立」忌；避开「内阁/大学士」等明清专名）。

社区声望阶梯（驱动指标 = 社交/创作行为，非游戏功名）：
| 阶 | 称号 | 解锁(社区声望) | 大致门槛 |
|---|---|---|---|
| 1 | 白身 | 0 | 注册 |
| 2 | 童生 | 50 | 完善档案/首登 |
| 3 | 生员 | 200 | 首次发布或收藏 |
| 4 | 举人 | 800 | 作品获赞/下载达标 |
| 5 | 贡士 | 2000 | 多作品被收藏 |
| 6 | 进士 | 6000 | 被社区推荐/高口碑 |
| 7 | 翰林 | 15000 | 资深创作者/策展 |
| 8 | 史官 | 40000 | 殿堂(接龙大成/被广泛 fork) |

声望值 `reputation = Σ 加权(发布×, 下载×, 获赞×, 获荐×, 接龙×, 被关注×)`，权重待调（反刷量：被荐/被 fork 权重高、易刷的下载权重低）。

> 决策点②：称号用**科举官制味**（白身→进士→史官）还是**中性**（见习→宗师）？官制味更沉浸、呼应游戏；但 CLAUDE.md 锁「chrome 朝代中立」。我判断科举词属世界观装饰可破例，但**要你拍**。
> 决策点③：档案默认可见性——公开 / 仅好友 / 私密？战绩是否默认公开？

### 数据契约
- 服务器 `users` 表扩展：`reputation INT, social_rank INT, bio TEXT, avatar_seed TEXT, profile_privacy TEXT`。
- 派生统计端点 `GET /profile?userId=`：聚合 作品数/下载和/获赞/粉丝/战绩摘要/成就/热力图(按天 count)。
- 战绩来源：需**通关/史册回流**（玩家发史册时附 `outcome/metrics`，见 C）。无遥测管线时，先只用「玩家主动发的史册」做战绩，不强求全自动。

## 三、A · 史馆动态流（内容层地基，最大杠杆）

参考 **小红书笔记流 + 微信朋友圈 + B站动态 + Steam 动态**。现 runtime **无 feed**，这是从 0 新建。

### 动态卡类型（邸报体，复用世态摘要层）
游戏已有 `buildWorldChangeDigest()`（世态变更摘要，读动态地理/世界变化，见记忆 `bot-世界态变更摘要层v1已落未ship`）——**正是自动生成动态卡的数据源**。
1. **局势高光**：截图 + 一句史笔 + 数值 delta。`天启七年冬·陕西民变平 ▸ 皇威 +12 · 民心 38→61`
2. **新作发布**：作者发了剧本/立绘/配乐 → 自动进关注者的流。
3. **史册落成**：一局通关 → 生成 AAR 卡（可接龙）。
4. **接龙邀请**：「谁来接着演这个残局？」
5. **里程碑**：升品 / 解锁成就 / 作品破千下载。

### 界面（单列沉浸 or 双列瀑布）
```
┌─ 史馆动态 ───────────────── [关注] [推荐] [本圈] ──┐
│ [印]史海钩沉 · 2h · 〔进士〕                        │
│  天启七年冬，陕西民变平。三月血战，皇威终立。      │
│  ┌──────────────┐  皇威 +12  民心 38→61            │
│  │  [局势截图]   │  存续 崇祯3年                    │
│  └──────────────┘                                  │
│  ♡ 86   评 12   荐 5   ↪ 引用接龙                  │
├────────────────────────────────────────────────────┤
│ [印]丹青手 发布了立绘包《盛唐人物》 …               │
└────────────────────────────────────────────────────┘
```

### 发布入口
- **游戏内**：回合末/通关时「记一笔入邸报」按钮 → 用世态摘要预填 → 玩家可编辑史笔 → 发。
- **工坊内**：作者主页/档案的「发动态」。
- feed 算法：`关注流`(纯时间线) + `推荐流`(热度×新鲜×多样性，冷启动给新作者流量池，对抗马太效应)。

### 数据契约
- 表 `posts(id, author_id, type, title, body, image_ref, refs_json, metrics_json, circle_id?, created_at, status)`；`post_likes(post_id,user_id)`；评论复用现有 `comments`（加 `target_type=post`）。
- 端点：`POST /feed/post` · `GET /feed?scope=following|recommend|circle&page=` · `POST /feed/like` · 举报复用。

## 四、C · 世界线接龙深化（护城河·只有天命能做）

现状：lineage(世界线树) + chronicles(史册接龙) + endorse。**深化三件**，参考 **GitHub fork·PR + TikTok 合拍/接龙 + 维基协作**。

### C-1 残局接演（接龙真正闭环）
史册节点带一个**可加载的残局存档指针**（不是整存档，是「剧本+关键世态 diff」的轻量种子）→ 别人「从这接着演」→ 游戏内 `史册→开局` 转换（蓝图 P3 缺口）→ 新史册自动挂为子节点。
```
天启原版 ──┬── 史海钩沉·中兴录 ──┬── 同志·守成篇 (偏安)
           │                      └── 边镇老卒·北伐篇 (进取)
           └── 你·袁崇焕不死 (架空)        ↑ 从「中兴录」残局接演
```

### C-2 同台竞史（擂台榜）
同一开局的「擂台」→ 玩家提交结局，按指标上榜：`存续年数 / 疆域 / 民心 / 皇威 / 国库`。参考 **抖音挑战赛 + 游戏排行榜**。
```
擂台·天启七年 ── 看谁的崇祯活最久
 ① 史海钩沉  崇祯17年·中兴   存续 17 · 疆域 +3
 ② 同志      崇祯12年·偏安   存续 12
 ③ 边镇老卒  崇祯9年·殉国    存续 9
```

### C-3 共编剧本（PR 式）
对他人剧本提**修订提案**（结构化 diff：改事件/数值/人物）→ 原作者审/并 → 记入世界线为「修订」分支。参考 **GitHub PR + 维基编辑**。MVP 可先只做「建议」（评论里附结构化建议），全 PR 流押后。

### 数据契约
- `chronicles` 扩展：`parent_save_ref, outcome, metrics_json`。
- `arenas(id, scenario_id, metric, title)` + `arena_entries(arena_id, user_id, chronicle_id, score)`。
- `revisions(id, pack_id, proposer_id, diff_json, status)`。
- 端点：`POST /chronicle/seed`(发残局) · `GET /chronicle/chain` · `POST /arena/submit` · `GET /arena?scenario=` · `POST /pack/revision`。

## 五、B · 创作者生态（留住作者）

参考 **B站 UP主 + Pixiv 作品集 + Steam 鉴赏家**。现 runtime 有作者页+endorse，**缺关注图谱/三连/合集/约稿**。

- **关注图谱**：关注作者(单向，区别于好友双向)→ 其新作/动态进我的 feed。`follows(follower_id, target_id)`。
- **一键三连**：收藏(star) + 赞 + 荐(endorse 已有)。收藏 `collections` 默认有个「我的收藏」。
- **鉴赏家合集**：任何人建「合集」策展 N 个剧本/史册（「明末入坑五部曲」「配乐电台·边关卷」）→ 合集本身可被关注/收藏。`collections(id,owner_id,title,kind)` + `collection_items(collection_id, pack_id|chronicle_id, ord)`。
- **约稿(后期·轻版)**：「给我的剧本求一套立绘/配乐」→ 需求墙 → 创作者私信接单。**只做撮合，不做钱**（守全平价 + 避支付/版权雷）。
- **作者主页** = 档案模板的公开视角（§二复用）。

## 六、D · 同好圈子（关系升级，偏重）

参考 **Discord 服务器·身份组 + Reddit 版块 + QQ群**。把弱关系升成归属感，但**最重**（要群+审核+治理）。

- **圈子** = 主题常驻社区：`明末研究会 / 配乐工坊 / 接龙擂台`。
- 圈内：过滤本圈的动态流 + 圈内榜单 + 置顶 + 活动赛。
- **身份组称号**：圈主/版主/资深；可挂社区品级（§二）。
- 审核：圈主自治 + 平台兜底（见 §八）。
- **MVP 建议「轻圈子」**：话题聚合页 + 加入 + 圈内 feed 过滤即可；群实时聊天/复杂角色/活动赛**押后**（实时 infra 比现 HTTP 重，蓝图 §五风险④）。
- 数据契约：`circles(id,name,topic,owner_id)` + `circle_members(circle_id,user_id,role)`。

## 八、风险与审核（社交必修，决定 D 为何押后）

1. **UGC 审核**：动态/合集/圈子贴都要进「举报-下架」闭环或先审。owner 已拍服务器自持审核（`project_workshop_account_sprint`）。
2. **反刷量**：声望权重要抗刷（被荐/被 fork 权重高，下载权重低）；注册门槛 + 限流。
3. **实时 infra**：圈子若做群聊 → 比 HTTP 重 → 这是 D 押后主因。
4. **隐私**：档案/战绩可见性开关（决策点③）。
5. **版权**：约稿不碰钱即避大半（决策点④）。

## 九、优先级与里程碑（建议分期）

> 原则：先把**「内容流 × 身份」最小闭环**跑起来（这是社交的地基），再上护城河，最后上重的关系层。沿用一刀一事、改前 `.bak`、node/playwright 验、不 ship 不 commit。

- **M1 · 脊梁 + 地基**（最小社交闭环）
  ① 个人资料库(史官档案，含社区品级/列传/收藏阁/成就，私+公双视角) ②A 动态流(关注流+局势高光卡，接世态摘要) ③B 关注/三连/收藏阁。
  → 形成「发动态 → 被看见 → 涨声望 → 档案变厚」的循环。**最高回报，先做。**
- **M2 · 护城河**：C 残局接演 + 同台竞史 + 鉴赏家合集。
- **M3 · 关系层**：D 轻圈子 + 共编剧本(PR) + 约稿需求墙。
- 每个 M 再拆刀；每刀都要 客户端(.tm-mall pane) + 服务器表/端点 + playwright 截图验。
