# 功名系统设计 · 2026-06（扩展自「功名机制」）

> owner 反馈：「当前人物功名机制简陋，全面优化升级、扩展为功名系统。」
> 朝代中立铁律：引擎层只放通用框架（出身路径枚举 / 派生量 / 1–18 级天花板）；具体科第名（进士/举人/生员…）作为**可剧本覆盖的默认预设**（科举时代通用阶梯），不写死单朝专名于不可覆盖处。

---

## 一、诊断：为什么「简陋」

实查（2026-06-13）锁定，当前游戏里「功名」其实是**两套各自为政、互不咬合**的东西：

1. **`virtueMerit`（功名=政绩标量）** — `char.resources.virtueMerit`（0~15000）+ `virtueStage`（1–6 阶：未识/有闻/清誉/儒望/朝宗/师表）。
   `tm-promotion.js` 已是**成熟的升迁脊柱**：18 级品级 floor、自动区（正四品及下）/政治区（从三品及上 level≤6）两层、科举按名次注入、失败减功名、越级强擢三档惩罚。这一块**不简陋**。

2. **`learning`（出身=惰性描述串）** — 剧本里 `learning:'进士'`、`'进士(翰林)'`、`'进士(状元)'`、`'举人'`……
   这是历史上功名的**灵魂**（出身资格），但在引擎里**毫无机械后果**：纯展示。「进士(翰林)」清流储相、直通阁部，与「举人」「捐纳」在游戏里享受**完全一样**的仕途天花板、名望底、党派归属。

**病根**：功名的「资格维度」（出身）被丢成一个死字符串，与「政绩维度」（merit）脱节。历史上一个官员的功名 = **出身（你凭什么入仕）× 政绩（你干出了什么）**，二者缺一不可。当前只建了后一半。

---

## 二、重构：功名 = 资格(出身) ⊕ 政绩(merit)

| 维度 | 含义 | 真源 | 变动性 |
|---|---|---|---|
| **出身（资格）** | 凭什么入仕：科举/门荫/纳赀/军功/吏进/荐辟/布衣，及科第等第、荣衔 | 新 `char.resources.gongming{}`（从 `learning` 解析回填，剧本可显式给） | 入仕时定，少变（可升监、改授、夺革） |
| **政绩（merit）** | 干出了什么：累积政绩、升迁凭据 | `char.resources.virtueMerit`（沿用） | 每回合涨跌（已成熟） |

「功名」对外 = 这二者的合体。图志里既显出身身份块，又显政绩阶。

## 三、出身结构化模型 `char.resources.gongming`

```
{
  path:    'keju'|'menyin'|'nazi'|'junggong'|'lijin'|'jianxuan'|'buyi',
           // 科举正途 / 门荫(荫叙) / 纳赀(捐纳) / 军功 / 吏进 / 荐辟征召 / 布衣(无功名)
  tier:    '进士'|'举人'|'贡生'|'监生'|'生员'|'武进士'|'武举'|'例监'|'吏员'|'荫生'|'',
           // 科第/资格等第名（剧本预设·可覆盖）
  honors:  ['翰林','庶吉士','状元','榜眼','探花','传胪','会元','解元','科道'...],
           // 荣衔（叠加修正·清流标记）
  tierRank: 0-100,   // 等第归一化高下（进士90·举人62·监生42·生员30…）— 排序/比较用
  zhengtu:  true|false,   // 正途 / 异途（派生：科举·正常军功·荐辟=正；纳赀·吏进·恩幸滥进=异）
  liupin:   'qing'|'mid'|'zhuo',  // 清流 / 中 / 浊流（派生：翰林科道=清；捐纳吏进恩幸=浊）

  youmian:  number,  // 个人优免额（免役丁数·派生：进士16·举人8·贡监6·生员2…）— 接赋税/户籍
  source:   'scenario'|'keju'|'menyin'|'nazi'|'junggong'|'lijin'|'edict'|'inferred',
  grantedTurn: number
}
```

- `庶吉士` → liupin=qing，ceiling→1，「储相」标记（清流领袖苗子）。

## 四、五大机械后果（「系统」的关键：功名必须 *做事*）

| # | 后果 | 接入点 |
|---|---|---|
| **A. 仕途天花板** | 出身 ceiling 限制可达品级。举人不得径入阁部（政治区 level≤6 被天花板挡）；玩家越级强擢突破天花板→在既有三档缺口惩罚上**叠加「出身不副」加重清议/名望罚**。 | `tm-promotion.js` 政治区 + 越级惩罚；endturn 自动升迁尊重 ceiling |
| **B. 名望底** | 清流（翰林/科道）抬名望底；异途（纳赀/恩幸）压名望底、成清议靶子。 | `char.resources.fame` 基线；图志名望 |
| **C. 党派/清流引力** | 进士清流→清流党（东林型）自然拉力；异途/恩幸→浊流党（阉党型）。接 Task1 阶层党派根基。 | `tm-social-foundation` / 党派归属 |
| **D. 个人优免** | 出身 youmian（免役丁）汇入赋税/户籍优免的**个人层面**（高功名→更多免役→更多投献依附→阶层连接）。 | 赋税/户籍优免（个人贡献项·轻量） |
| **E. 同年/座师网** | 同科同年→同年之谊；主考→门生（座师）。〔后续 slice·轻量接既有关系〕 | 关系系统 |

A–D 为本轮主体，E 视余量。

## 十、本轮如实标注的边界

- **优免（后果 D）落到出身级**：每员个人优免额（免役丁）已建模、图志可视、`totalYoumian` 可聚合。但赋税/户籍引擎**当前无量化优免扣减机制**（优免在阶层级是定性 `privilege` 标签）。把个人功名优免接成真实税基扣减需**另起赋税侧机制**，属独立赋税特性、不在功名系统范围内擅自侵入——本轮留 `totalYoumian` 接口 + 出身级可视，量化扣减待单列。
- **党派/清流引力（后果 C）走 AI 面**：`liupinAffinity` 纯函数暴露 + AI prompt 注入清流/异途，让 AI 党派归属顺出身——未硬改 Task1 党派引擎（避免重审侵入），靠 AI 演绎落地。
- **同年/座师网（后果 E）**：本轮未做，留后续。
- **政绩（merit）半边**：沿用既有 `virtueMerit`，图志「政绩」栏忠实显示其值（含历史档可能的低/负值，属既有 merit 系统范畴，非本系统计算）。
